Simulacion

Primero, antes de pensar en trabajar con componentes fisicos, preparamos el codigo que teniamos que utilizar con componentes ficticios en una simulacion. Utilizando el lenguaje de arduino que es un tipo de mezcla entre c++/java y c mas las funciones de "void loop()" y "void setup()". Con el programa Proteus, pudimos probar el codigo mientras lo haciamos, armando el circuito que ibamos a tener en un futuro en fisico, para simplemente correr el codigo y ver si hace lo que se supone que deberia hacer.

Construccion

Una vez que la simulacion fue un exito, armamos el arduino, utilizando un protoboard para comunicar el arduino con los componentes. El protoboard lo utilizamos como un puente para conectar el LCD y los botones matriciales al arduino.
